# 2026-01-19 to 2026-01-25

Source: https://kb.kentik.com/docs/2026-01-19-to-2026-01-25

---

## AI / Insights

### Features

* **Gemini Flash 3 Preview model**: Added support for the Gemini Flash 3 Preview model, giving users access to the latest Google AI capabilities within Kentik AI Advisor.
* **MCP Server deployment**: The AI Advisor MCP (Model Context Protocol) server is now deployed as a Kentik service and exposed to the internet, enabling external AI tools and agents to interact with Kentik AI Advisor programmatically.
* **Dynamic per-company investigation limits**: AI Advisor auto-investigation limits are now configurable per company through the portal, replacing static environment-level overrides for more flexible control.
* **AI Advisor device config tool**: Updated the AI Advisor device configuration tool toggle with improved copy for clarity.

### Bug Fixes

* **AI Advisor stability improvements**: Fixed internal state errors and improved stability of the AI Advisor service.
* **MCP Server request size enforcement**: Added a maximum request size limit to the MCP server to prevent oversized requests from causing issues.

## Alerting

### Features

* **Backfill query optimization**: Improved alerting backfill performance by matching query limits with policy limits when keyed by device, significantly speeding up backfill jobs.

### Bug Fixes

* **Policy description retention**: Fixed an issue where alert policy descriptions were not being retained after updates.
* **Context search fix**: Resolved a bug affecting context-based search within alerting.
* **Subpolicy management improvements**: Fixed multiple subpolicy issues including notification channels not saving, frequency changes not persisting, and dimension grouping problems.
* **MKP fix**: Resolved an issue with MKP (Managed Kentik Portal) alerting behavior.

## Cloud

### Features

* **Azure VNet flow logs support**: Added support for Azure Virtual Network (VNet) flow logs, replacing the deprecated NSG flow logs in preparation for Microsoft's retirement of NSG flow logs.
* **AWS security group enrichment with IPv6 support**: Enhanced AWS flow data enrichment to include security group information, with support for IPv6 addresses.

### Bug Fixes

* **Azure flow timestamps**: Fixed incorrect timestamps on Azure flow log data.
* **OCI compartment fallback**: Resolved an issue with OCI (Oracle Cloud Infrastructure) compartment resolution fallback logic.
* **Azure export editing**: Fixed bugs preventing users from editing existing Azure cloud exports.

## Core

### Features

* **Resizable table columns**: Tables across the product now support resizable columns, allowing users to adjust column widths to fit their content. Column sizes are persisted across sessions.
* **SSE real-time infrastructure**: Introduced Server-Sent Events (SSE) infrastructure for real-time data streaming in the UI, enabling live data updates without page refreshes.

### Bug Fixes

* **Email subject fix**: Fixed an issue where HTML tags were incorrectly appearing in email notification subjects.
* **Tag case sensitivity & performance**: Fixed tag lookup to be case-sensitive as intended, resolving internal state errors and reducing CPU overhead caused by duplicate populators.
* **Query reliability & speed improvements**: Improved the reliability and performance of flow data queries through internal pipeline optimizations.
* **Lossless flow data during device updates**: Fixed an issue where flow data could be lost during device configuration updates (edating), ensuring no data is dropped during the process.
* **NMS sidebar filter persistence**: Fixed a bug where NMS sidebar filters were not persisting correctly between page loads.

## Network Monitoring System (NMS)

### Features

* **SNMP polling enabled by default**: SNMP polling is now enabled by default for new devices, reducing manual configuration steps for customers onboarding NMS.
* **Dynamic NMS measurements**: Added support for dynamic measurement configuration in NMS, allowing more flexible metric collection.
* **Versa IPv4 support**: Added IPv4 support for Versa Networks devices in NMS SNMP profiles.
* **Agent distro compatibility fallback**: Implemented a distribution compatibility fallback system for Universal Agent, allowing agents to fall back to a compatible package when their exact Linux distribution is not supported.
* **Infoblox DNSTAP support**: Added DNSTAP measurement support for Infoblox devices.

### Bug Fixes

* **NetApp metrics fix**: Resolved an issue with incorrect or missing metrics for NetApp devices.
* **Agent capability health reporting**: Fixed a bug where agent capability health was incorrectly reporting `Unreachable` status as healthy.
* **MIB Browser dark mode**: Fixed display issues in the MIB Browser when using dark mode.

## Synthetics

### Features

* **Synth AI parent test results**: AI Advisor can now surface parent-level Synthetics test results, providing higher-level visibility into test health when queried through AI Advisor.

### Bug Fixes

* **Dashboard panel display fixes**: Fixed multiple display issues with Synthetics panels in dashboards and library exports, including incorrect heights, missing loading indicators, and cut-off content in availability tests.
* **Test enrichment data**: Fixed an issue where test enrichment data was not being correctly populated.
* **Targets filter fix**: Resolved a bug where the targets filter in Synthetics was not functioning correctly.
* **DNS timeout editing**: Fixed an issue preventing users from editing DNS timeout settings on Synthetics tests.
